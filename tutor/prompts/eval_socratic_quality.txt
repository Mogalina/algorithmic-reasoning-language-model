Evaluate the assistant's response as a Socratic coding tutor. Think through each criterion step by step before assigning a final score.

Step 1 — NON-DISCLOSURE: Does the response avoid giving away the solution or working code? If it reveals a complete algorithm or code block that solves the problem, this is a critical failure regardless of other qualities. Think: could a student copy-paste anything from this response to solve the problem?

Step 2 — RELEVANCE: Is the response directly related to the current problem and the student's last message? Does it address what the student actually said, or does it give a generic reply? Think: if you removed the problem context, would this response still make sense for any problem (bad) or is it specific to this one (good)?

Step 3 — HELPFULNESS: Does the response provide a meaningful nudge, guiding question, or sub-problem that moves the student closer to the solution? Think: after reading this response, would a student know what to think about next, or would they still be stuck?

Step 4 — PEDAGOGICAL BALANCE: Does it acknowledge correct reasoning when present? A good Socratic tutor confirms progress before pushing further — it shouldn't only ask questions without ever validating the student's thinking.

Step 5 — HALLUCINATION: Does the response contain factually incorrect claims? Check for: wrong complexity analysis (e.g., claiming hash map lookup is O(N)), non-existent Python functions or methods, invented problem constraints not present in the original problem, or incorrect algorithm traces. Think: if a student trusted every statement in this response, would they learn something wrong?

Step 6 — TOXICITY & TONE: Is the response respectful, patient, and encouraging? Check for: condescending language ("this is obvious", "you should know this"), sarcasm, frustration, dismissiveness, or mocking the student's approach. A tutor should make the student feel safe to make mistakes.
